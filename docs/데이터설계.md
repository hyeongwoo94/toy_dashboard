프로젝트 진행 순서
1. 내가 다룰 데이터 구조를 정의한다.
    1-1 기준별 모듈 상태 정의
        - 도메인 설계
            [task 모듈]
                - id
                - 제목
                - 작성 날짜
                - 마감일
                - 업무 내용
                - 업무 중요도
                - 상태(해야하는일, 진행중, 끝)
                - 작성자
                - 담당자
            [유저 모듈]
                - password
                - loginId(로그인할 때 작성하는 아이디)
                - name(실제 유저 이름)
                - role(권한)
        - 전역 상태 설계
            [권한 모듈]
                - id
                - name
                - role
                - isLoggedIn(로그인 유무)                
        - UI 상태 기준
                - 로그인 권한에 따른 보여줄 ui
                - 모달 열리고 닫힘에 따른 ui

2. 설계한 구조 코드로 옮기기
    - 응집도와 의존성을 항상 생각하면서 파일 만들기
    - src/type/ 파일들 생성

3. 상태를 어디에 둘 것인가?
    - 각각의 상태를 각각의 집에서 컨트롤하기
    - 스토어 안에는 어떤 내용이 들어가야 할까?
        - 해당 스토어가 갖고있는 상태 값
        - 상태를 바꿔주는 함수들
4. 각 store가 갖고 있어야하는 액션은 무엇인가?
    - auth
        -로그인/비로그인
    - task
        -CRUD 액션
    - user
        - 관리자가 권한을 지정해주는 액션
